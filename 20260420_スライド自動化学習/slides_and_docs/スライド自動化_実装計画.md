# スライド画像生成の計画

いただいたご要望に沿ってスライドの画像生成を行うための実装計画です。

## ⚠️ User Review Required（重要な確認事項）

今回ご依頼いただいた画像生成において、環境の制限により以下の2つの重要な課題がございます。ご確認とご判断をお願いいたします。

1. **「NanoBananaPro」というツールへのアクセスがありません**
   私には当該の名称のツールが存在しないため、代わりに私の**内蔵画像生成AI機能**を使用して画像を作成することになります。
2. **AI画像生成では「正確な日本語テキスト」を出力できません**
   現在の画像生成AIは、英語の短い単語以外の文字を正確に描画することが非常に苦手です。そのため、指定されたスライドの日本語テキストをそのまま画像に書き込もうとすると、**解読不能な文字（文字化けのような状態）**になってしまいます。「文字を清書して再生成する」という処理も技術的に対応が困難です。

## 提案する代替アプローチ

上記の制限を回避し、できる限りご要望に近い「付箋ノート風の16:9スライド」を作成するために、以下のいずれかのアプローチをご提案します。

### 【Option A】背景デザインのみを画像生成し、文字は後からユーザー様側で乗せる
画像生成AIを使って「学生のノート、付箋、蛍光ペン」を用いたオシャレな16:9スライドの背景・図解のベースだけを7枚並列生成します。文字部分はダミーの記号のようなものになるか、あえて文字なしで生成します。生成後、指定された `Outputs/YYYY-MM-DD-SS/` ディレクトリに画像を保存（コピー）します。
（ご自身でPowerPointやCanva等に背景として取り込み、テキストを重ねる使い方になります）

### 【Option B】HTML/CSSで「付箋ノート風のデザイン」を作り、スクショを撮る（文字が正確）
AIの画像生成ツールを使わず、私が**HTMLとCSSを使ってブラウザ上で動く「付箋ノート・蛍光ペン風のスライド」をコーディング**します。その後、私がブラウザを操作して各スライドのスクリーンショット（2K・16:9）を撮影し、指定のディレクトリに保存します。
（この方法であれば、**日本語のテキストが完璧に正確な状態で、学生ノート風のデザインのスライド画像**を取得できます）

---

## 予定している作業フロー（Option B の場合）

1. `Outputs/YYYY-MM-DD-SS` ディレクトリをPowershellコマンドで新規作成。
2. `scratch/slide_preview.html` を作成し、ノート風の背景、付箋、蛍光ペン風のハイライトを実装したCSSコーディングを行う。
3. 同HTMLに7枚分のスライド構造（16:9のアスペクト比固定、2K相当のサイズ）を持たせ、提示したMarkdownのテキストを入れ込む。
4. `browser_subagent` を起動し、そのHTMLを開いて7枚のスライドをそれぞれスクリーンショット撮影する。
5. 保存されたスクリーンショットを `Outputs/...` に移動し、各スライドのタイトル名にリネームする。
