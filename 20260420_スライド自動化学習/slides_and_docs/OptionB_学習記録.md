## 🛠 詳細な作業手順

今回のOption Bを構成した技術的なステップは以下の通りです。

### 1. 画面キャプチャ環境（Playwright）の構築
ブラウザを自動操作し、解像度を指定して正確なスクリーンショットを撮影するために、Pythonライブラリ「Playwright」を利用しました。
以下のコマンドでパッケージとバックグラウンド用ブラウザ（Chromium）を導入しました。
```bash
python -m pip install playwright
python -m playwright install chromium
```

### 2. Pythonスクリプト（`generate_slides.py`）の作成
指定された7つのスライドの情報（HTML）とデザイン（CSS）を内包し、順次画像を生成・保存する自動化プログラムを作成しました。

**🎨 適用したCSSデザインのポイント**
* **大学ノート風の背景**: CSSグラデーションを用いて罫線柄と、ノート左端のバインダー用の穴を表現。
* **付箋や文房具感の設定**: 要素を傾けたり(`transform: rotate()`)、影(`box-shadow`)や折り目をつけて付箋（Post-it）風に配置。
* **蛍光ペンのハイライト**: 背景色を直線的に敷くことで、文字に蛍光ペンで線（`<mark>`やカスタムクラス）を引いたような効果（`linear-gradient`）を再現。

**⚙️ スクリプトの処理フロー**
1. スクリプト実行日時に基づく格納ディレクトリを設定（`Outputs/2026-04-20-...`等）。
2. `temp_slides.html` という一時ファイルとしてHTMLを書き出し。
3. Playwrightでバックグラウンドブラウザを `viewport={'width': 2560, 'height': 1440}`（2K・16:9）で起動。
4. ローカルの `temp_slides.html` を読み込み、Webフォントの読み込み完了を待機。
5. ページ全体のスクリーンショットを撮影し、画像として保存。これを7枚分繰り返す。
6. 一時HTMLファイルを削除して処理完了。

### 3. スクリプトの実行と画像の保存
上記で作成したスクリプトを実行（`python generate_slides.py`）し、一括で7枚の画像ファイルを自動撮影して出力しました。
